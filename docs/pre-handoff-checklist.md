# Pre-handoff Checklist

Что проверить перед передачей админки клиенту.

## Админка

- [ ] Открывается `https://<client>.vercel.app` без ошибок
- [ ] Дашборд показывает данные (не нули):
  - [ ] Выручка
  - [ ] Количество чеков
  - [ ] Средний чек
  - [ ] Остатки на складе
- [ ] Логин работает (если `VITE_REQUIRE_AUTH=***)
- [ ] Все разделы открываются:
  - [ ] Аналитика
  - [ ] Кассовые смены
  - [ ] Чеки
  - [ ] Блюда и категории
  - [ ] Ингредиенты
  - [ ] Сотрудники
  - [ ] Схема зала
  - [ ] Поставки / Списания / Перемещения
  - [ ] Инвентаризация

## Инвентаризация

- [ ] Создать новую инвентаризацию (Кухня → Полная)
- [ ] Колонки **Поступление** показывают не нули
- [ ] Колонки **Расход** показывают не нули
- [ ] Если данных нет (новый ресторан) — колонки нули, но таблица не ломается

## POS

- [ ] Открывается на планшете
- [ ] Экран блокировки → ввод PIN
- [ ] Открытие смены
- [ ] Создание заказа: добавить блюдо → модификаторы → количество
- [ ] Оплата: наличные / карта
- [ ] Закрытый чек появляется в «Закрытые заказы»
- [ ] После оплаты — проверить админку:
  - [ ] В чеках появился новый paid заказ
  - [ ] В inventory_movements появилась запись расхода

## Склад

- [ ] Создать поставку → проверить что `stock_items` обновился
- [ ] Создать списание → проверить что остаток уменьшился
- [ ] Создать перемещение между складами

## Быстрая проверка SQL

```sql
-- Все ли ключевые таблицы существуют
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public'
AND table_name IN (
  'orders', 'order_items', 'payments', 'shifts',
  'products', 'categories', 'recipe_items',
  'warehouses', 'workshops', 'stock_items',
  'inventory_movements', 'warehouse_deliveries',
  'warehouse_write_offs', 'warehouse_transfers'
);

-- Есть ли venue
SELECT id, name FROM venues;

-- Работают ли RPC
SELECT admin_inventory_period_movements(
  (SELECT id FROM venues LIMIT 1),
  (SELECT id FROM warehouses LIMIT 1),
  '2024-01-01'::timestamptz,
  '2030-01-01'::timestamptz
);
```
